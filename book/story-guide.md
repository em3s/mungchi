# 쌍둥이별 이야기 — 집필 지침서

이 문서는 "쌍둥이별 이야기" 전자책을 쓰고 유지하기 위한 규칙을 정의한다.
책 본문은 `docs/chapters/` 안의 MDX 파일이고, 이 지침서는 그 파일들을 쓰는 방법을 설명한다.

---

## 1. 독자

- **초등학교 3학년** (만 9세)
- 한글을 유창하게 읽지만, 긴 문장은 힘들어함
- 이모지와 그림에 강하게 반응
- 자기 이름이 나오면 집중력 급상승

---

## 2. 문체 규칙

### 문장
- 한 문장 **20자 이내** 권장, 최대 30자
- 한 문단 **3~4문장** 이내
- 존댓말 (해요체): "~했어요", "~거예요", "~볼까요?"
- 어려운 한자어 금지: 각성(X) → 눈뜨다(O), 초월(X) → 넘어서다(O)
- 의성어/의태어 적극 사용: 반짝반짝, 두근두근, 쑥쑥

### 피해야 할 것
- 수동태: "빛나게 되었다" → "빛나기 시작했어요"
- 이중 부정: "안 할 수 없다" → "해야 해요"
- 설명조: "이것은 ~라는 뜻입니다" → 이야기로 보여주기
- 교훈 직접 말하기: "그래서 매일 해야 해요" → 캐릭터 행동으로 보여주기

### 좋은 예시
```
별이 깜빡깜빡 빛났어요.
시현이가 할일을 하나 끝낼 때마다,
별이 조금씩 더 밝아졌거든요.

"오늘도 하나 했다!"
시현이가 웃으며 별을 올려다봤어요.
```

### 나쁜 예시
```
할일을 완료하면 달성률이 올라가고 달성맵의 노드가 해금됩니다.
이는 게이미피케이션을 통한 습관 형성 메커니즘의 일부입니다.
```

## 3. 세계관

### 핵심 설정
- 밤하늘에 아직 빛나지 못하는 별들이 있음
- 별이 빛나려면 매일의 작은 행동(할일, 공부, 습관)이 에너지로 쌓여야 함
- **"매일 하나씩 하면, 별이 빛난다"** — 이야기 전체를 관통하는 한 줄

### 캐릭터

| 이름 | 별 | 역할 | 성격 키워드 | 말투 |
|---|---|---|---|---|
| 시현 | ⭐ 반짝별 | 수호자 | 꾸준함, 책임감, 조용한 자신감 | 차분, "...해볼게" |
| 미송 | ✨ 반짝별 | 탐험가 | 호기심, 용기, 에너지 | 활발, "이거 해보자!" |
| 아빠 | 🛡️ 든든별 | 이끔이 | 듬직함, 유머 | "걱정 마, 아빠가 있잖아" |
| 엄마 | 💖 따뜻별 | 지킴이 | 따뜻함, 응원 | "잘하고 있어, 조금만 더" |

### 캐릭터 관계
- 시현 ↔ 미송: 서로 다른 방식으로 별을 키움. 경쟁이 아니라 협력.
- 부모: 직접 별을 키우지 않지만 길을 만들어주고 응원함.
- 쌍둥이별: 두 아이의 별이 합쳐진 것. 혼자보다 둘이 함께할 때 더 크게 빛남.

### 사용하면 안 되는 표현
- "공부해야 해", "안 하면 안 돼" 같은 강제/압박 표현
- 시현이와 미송이를 비교하는 표현
- 실패를 부정적으로 묘사 (못 한 날 = 별이 쉬는 날)

## 4. 책 구조

### 4막 구조

달성맵의 4개 Act를 책의 4개 파트로 대응.

| 파트 | 달성맵 Act | 테마 | 챕터 수 (예상) |
|---|---|---|---|
| 1부: 두 별의 시작 | Act 1 (0~200) | 첫 빛, 습관의 씨앗 | 3~4 |
| 2부: 두 별이 만나다 | Act 2 (300~800) | 발견, 함께의 힘 | 3~4 |
| 3부: 쌍둥이별의 눈뜸 | Act 3 (1000~2500) | 성장, 전설이 되다 | 3~4 |
| 4부: 쌍둥이별의 여행 | Act 4 (3000~5000) | 모험, 완성 | 3~4 |

### 챕터 파일 규칙
- 파일명: `NN-slug.mdx` (예: `01-two-stars.mdx`)
- 하나의 챕터 = 스크롤 기준 3~5 화면 분량
- 챕터 시작은 반드시 `<ChapterHeader>` 컴포넌트
- 챕터 끝은 다음 챕터로의 기대감을 남기는 문장

### 챕터 잠금 (앱 연동)
- 각 챕터에 `unlockAt` 값 (달성맵 노드 번호)
- 해당 노드에 도달해야 읽을 수 있음
- 잠긴 챕터는 제목만 보이고 🔒 표시

## 5. MDX 컴포넌트

책 본문에서 사용할 수 있는 커스텀 컴포넌트 목록.

### `<ChapterHeader>`
챕터 제목. 모든 챕터의 첫 요소.
```mdx
<ChapterHeader emoji="🚀" title="빛나지 못하는 별" act={1} chapter={1} />
```

### `<CharacterCard>`
캐릭터 소개 또는 대사 카드.
```mdx
<CharacterCard name="시현" emoji="⭐" role="반짝별 수호자">
  "오늘도 하나 했다!"
</CharacterCard>
```

### `<Highlight>`
강조 문장 (인용문, 핵심 메시지).
```mdx
<Highlight>매일 하나씩 하면, 별이 빛난다.</Highlight>
```

### `<Progress>`
독자의 실제 진행도 삽입 (앱 데이터 연동).
```mdx
<Progress label="나의 별은 지금..." />
```
